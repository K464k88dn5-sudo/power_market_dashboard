## 一、项目定位

为电力市场从业者提供一个**实时、多源、可视化**的电力市场运行监控平台，集成气象、燃料、电价、检修四大核心数据模块，支持电价预测模型推理，服务于电力现货交易决策。

- **禁止 emoji 图标**：标题、按钮、卡片使用文字标签
- **禁止厚重投影**：不使用 `box-shadow > 0 16px 32px`
- **禁止花哨动画**：不使用弹跳、旋转、扫描线
- **禁止旋转文字**：横坐标水平显示

## 四、数据模块规格

### 4.1 气象监测

| 子模块 | 数据源 | 刷新周期 | 展示形式 |
|--------|--------|---------|---------|
| 逐时温度 | Open-Meteo API | 5分钟 | 折线图（当日24h） |
| 温度趋势 | Open-Meteo API | 5分钟 | 多线图（最高/均温/最低） |
| 风速 | Open-Meteo API | 5分钟 | 填充折线图 |
| 湿度 | Open-Meteo API | 5分钟 | 填充折线图 |
| 实时观测 | wttr.in | 5分钟 | 数值卡片 |

**API 端点：**
- 预报：`https://api.open-meteo.com/v1/forecast`
- 实时：`https://wttr.in/{城市}?format=j1`

### 4.2 燃料价格

| 子模块 | 数据源 | 刷新周期 | 展示形式 |
|--------|--------|---------|---------|
| 动力煤价格 | CCTD 环渤海港口 API | 1小时 | 折线图 |
| LNG 出厂价 | 上海石油天然气交易中心 | 1小时 | 折线图 |

**API 端点：**
- 煤价：`http://www.cctd.com.cn/Echarts/data/HBHCKJ_DLMQH.php`
- LNG：`https://www.chinacqpgx.com/cyl?pr=GDLNG`

### 4.3 广东地市实时温度

| 子模块 | 数据源 | 刷新周期 | 展示形式 |
|--------|--------|---------|---------|
| 温度地图 | Open-Meteo + GeoJSON | 5分钟 | Folium 热力地图 |
| 统计摘要 | 计算值 | 实时 | 文字标签 |

**技术规格：**
- 地图底图：`CartoDB positron`（浅色）
- GeoJSON：21个广东地市，含 `adcode`、`centroid`
- 色阶：`LinearColormap(vmin=18, vmax=40)`
- 填充透明度：0.5
- 城市标注：白色文字 + 黑色阴影

### 4.4 检修计划

| 子模块 | 数据源 | 刷新周期 | 展示形式 |
|--------|--------|---------|---------|
| 机组检修 | 信息披露 xlsx | 按需 | HTML 表格（滚动） |
| 输变电检修 | 信息披露 xlsx | 按需 | HTML 表格（滚动） |
| 检修容量 | 信息披露 xlsx | 按需 | 数值标签 |

**数据目录：**
- 本地：`~/Desktop/能源电力资料/日前训练数据/信息披露日前/`
- 云端：`disclosure/`（同步脚本自动复制最近7天）

### 4.5 电价分析

| 子模块 | 数据源 | 刷新周期 | 展示形式 |
|--------|--------|---------|---------|
| 实际电价 | 日前节点电价.xlsx | 按需 | 24h 折线图 |
| 预测电价 | 广东日前电价预测.xlsx | 按需 | 叠加折线图 |
| 模型推理 | forecast/predictor.py | 按需 | 实时预测 |
| 峰谷指标 | 计算值 | 实时 | 数值标签 |
| 热力图 | 日前节点电价.xlsx | 按需 | Heatmap (24h×30d) |

**电价图特性：**
- 峰谷区间着色：谷时(0-6)蓝底、峰时(8-12,14-17)红底、尖峰(10-12,15-17)深红底
- 当前时刻标记线（热力图）
- 历史数据对比箭头

### 5.2 核心依赖

```
streamlit>=1.50

streamlit-folium>=0.18
```

### 5.3 缓存策略

| 数据 | TTL | 缓存机制 |
|------|-----|---------|
| 气象数据 | 300秒 | `@st.cache_data(ttl=300)` |
| 燃料数据 | 3600秒 | `@st.cache_data(ttl=3600)` |
| 电价数据 | 300秒 | `@st.cache_data(ttl=300)` + 文件缓存 |
| 电价预测 | 永久 | 文件缓存 `predictions/` |

### 5.4 错误处理

- **网络不可达**：`socket` 预检 → 直接使用缓存/模板
- **数据源失败**：自动降级到参考模板
- **BrokenPipeError**：所有 `print()` 替换为 `logging`
- **数据缺失**：显示友好提示，不崩溃

## 六、交互功能

### 6.1 数据上传

- 侧边栏：电价数据、检修数据文件上传
- 电价分析模块：信息披露文件上传

### 6.2 数据同步

- 按钮：「同步公网」（电价分析模块右侧）
- 功能：运行 `sync_data.sh`，推送数据到 GitHub
- 定时：launchd 每天 09:00 自动执行

### 6.3 日期选择

- 日期选择器：中文格式（如"6月10日 周二"）
- 默认：最近有实际数据的日期
- 联动：检修计划跟随电价分析日期

### 6.4 电价预测

- 触发：选择日期 → 点击预测按钮
- 模型：LGB 20% + XGB 20% + CatBoost 60%
- 输出：24h 预测电价曲线
- 缓存：已预测日期秒级加载

## 八、性能优化

| 优化项 | 方案 | 效果 |
|--------|------|------|
| 图表渲染 | Scattergl (WebGL) | 大数据量 5-10x 提速 |
| 预测缓存 | 文件缓存 + 预加载 | 已预测日期秒级加载 |
| 网络韧性 | socket 预检 + 降级 | 网络异常不崩溃 |
| 数据缓存 | st.cache_data | 减少重复请求 |

## 十、已知限制

1. **地图**：folium 无 GPU 加速，大数据量时较慢
2. **电价数据**：本地 xlsx 文件，公网需手动同步
3. **预测模型**：依赖本地 `~/projects/gd-price-forecast/` 项目
4. **检修数据**：依赖本地信息披露文件目录
5. **Streamlit Cloud**：无法访问本地文件，需通过上传或同步
